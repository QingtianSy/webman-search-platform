# 产品需求文档（PRD）

## 1. 产品定位

### 1.1 产品名称
Webman Search Platform（题库搜题 SaaS 平台）

### 1.2 一句话描述
面向教育机构和开发者的高性能题库搜题平台，提供 Web 端搜题和 API 开放接口两种服务形态。

### 1.3 目标用户

| 用户类型 | 描述 | 核心诉求 |
|---------|------|---------|
| 普通用户 | 学生、教师、自学者 | 快速搜题、查看答案解析 |
| 机构用户 | 教育机构、培训公司 | 批量搜题、API 对接、数据统计 |
| 开发者 | 第三方平台开发者 | API 接入、SDK 集成 |
| 运营人员 | 平台内部运营 | 题库管理、用户管理、数据分析 |
| 管理员 | 平台管理者 | 系统配置、权限管理、全局监控 |

### 1.4 商业模式

```
┌─────────────────────────────────────────────┐
│              收入来源                         │
├──────────────┬──────────────┬───────────────┤
│  免费套餐     │  付费套餐     │  API 调用      │
│  每日 10 次   │  月/年订阅    │  按次计费      │
│  基础功能     │  高级功能     │  开放平台      │
└──────────────┴──────────────┴───────────────┘
```

### 2.2 两端一体

| 维度 | 用户端 | 管理端 |
|------|-------|-------|
| 入口 | 统一登录，按角色跳转 | 统一登录，按角色跳转 |
| 认证 | JWT + UserAuthMiddleware | JWT + AdminAuthMiddleware |
| 菜单 | 按 permission_code 动态渲染 | 按 permission_code 动态渲染 |
| 数据 | 只看自己的数据 | 看所有用户的数据 |

## 3. 核心业务流程

### 3.1 搜题主流程（核心闭环）

```
用户输入关键词
    │
    ▼
前端发起 POST /api/v1/user/search/query
    │
    ▼
UserAuthMiddleware 验证 JWT
    │
    ▼
SearchService 检查用户额度（Redis）
    │  ├─ 额度不足 → 返回错误
    │  └─ 额度充足 → 继续
    ▼
Redis 查搜索缓存
    │  ├─ 命中 → 直接返回
    │  └─ 未命中 → 继续
    ▼
Elasticsearch 全文检索
    │  └─ 返回 question_id 列表
    ▼
MongoDB 批量查询完整题目
    │
    ▼
Redis 扣减额度 + 缓存结果
    │
    ▼
MySQL 写搜索日志索引
MongoDB 写搜索日志详情
    │
    ▼
返回搜索结果给前端
```

### 3.2 用户注册登录流程

```
用户提交用户名/密码
    │
    ▼
AuthService 验证凭证（MySQL users 表）
    │  ├─ 失败 → 返回错误
    │  └─ 成功 → 继续
    ▼
查询用户角色和权限（MySQL RBAC 表）
    │
    ▼
JwtService 签发 Token
    │
    ▼
Redis 缓存 Token + 权限 + 菜单
    │
    ▼
返回 Token + 用户信息 + 菜单树
    │
    ▼
前端按 default_portal 跳转到用户端或管理端
```

### 3.3 API 开放平台调用流程

```
第三方发起 POST /open/v1/search/query
    │  Header: X-Api-Key: xxx
    ▼
OpenApiAuthMiddleware 验证 API Key（Redis）
    │  ├─ 无效 → 401
    │  ├─ IP 白名单校验失败 → 403
    │  └─ 有效 → 继续
    ▼
检查 API Key 绑定的额度
    │
    ▼
执行搜题（同主流程）
    │
    ▼
记录 API 调用日志
    │
    ▼
返回 JSON 结果
```

### 3.4 题目入库流程

```
管理员上传/导入题目
    │
    ▼
QuestionService 数据校验与清洗
    │
    ▼
MongoDB 写入完整题目（questions 集合）
    │
    ▼
QuestionIndexService 同步到 Elasticsearch
    │  └─ 提取搜索字段，建立倒排索引
    ▼
MySQL 更新题目统计（分类、来源、标签计数）
    │
    ▼
Redis 清除相关搜索缓存
```

### 3.5 采集任务流程

```
管理员创建采集任务
    │
    ▼
CollectService 初始化任务（MySQL collect_tasks）
    │
    ▼
异步 Worker 执行采集
    │  ├─ 调用第三方 API 源
    │  ├─ 原始结果写入 MongoDB（collect_raw_results）
    │  └─ 更新任务进度（Redis）
    ▼
数据清洗与去重
    │  └─ MD5 去重
    ▼
入库（MongoDB + ES 同步）
    │
    ▼
任务完成，更新状态
```

## 5. 非功能需求

### 5.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 搜题接口响应时间 | P99 < 200ms |
| 列表接口响应时间 | P99 < 500ms |
| 并发搜题 QPS | ≥ 1000 |
| 系统可用性 | ≥ 99.9% |

### 5.2 安全要求

| 要求 | 说明 |
|------|------|
| 认证 | JWT Token，7 天过期 |
| 授权 | RBAC 角色权限控制 |
| 传输 | HTTPS（Nginx 层） |
| 密码 | bcrypt 加密存储 |
| 限流 | Redis 令牌桶，防刷 |
| 注入 | 参数化查询，防 SQL/NoSQL 注入 |
| XSS | 输出转义 |
| 敏感数据 | 密钥不入库，env 管理 |

### 5.3 可扩展性要求

| 要求 | 说明 |
|------|------|
| 数据源切换 | config/integration.php 控制 mock/real |
| 多 API 源 | 支持配置多个第三方搜题 API |
| 水平扩展 | Webman 多 Worker 进程 |
| 未来拆分 | 按业务中心可拆为微服务 |
