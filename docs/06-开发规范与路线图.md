# 开发规范

## 1. 后端代码规范

### 1.1 Controller 规范

```php
<?php
namespace app\controller\admin;

use app\service\admin\QuestionAdminService;
use app\validate\admin\AdminQueryValidate;
use support\ApiResponse;
use support\Request;

class QuestionController
{
    // ✅ 正确写法：接收 Request，调用 Validate → Service → ApiResponse
    public function index(Request $request)
    {
        $query = (new AdminQueryValidate())->list($request->get());
        return ApiResponse::success((new QuestionAdminService())->getList($query));
    }
}
```

**必须遵守**：
1. 方法第一个参数必须是 `Request $request`
2. 参数校验交给 Validate 层
3. 业务逻辑交给 Service 层
4. 统一用 `ApiResponse::success()` / `ApiResponse::error()` 返回
5. Controller 不写业务逻辑，只做编排
6. **禁止**在 Controller 中直接返回硬编码 JSON

### 1.2 Service 规范

```php
// 列表方法统一返回格式
public function getList(array $query = []): array
{
    return [
        'list'      => [],
        'total'     => 0,
        'page'      => $query['page'] ?? 1,
        'page_size' => $query['page_size'] ?? 20,
    ];
}

// 写操作统一返回格式
public function create(array $data): array
{
    return [
        'success' => true,
        'action'  => 'create',
        'id'      => $id,
        'data'    => $row,
    ];
}
```

### 1.3 Repository 规范

```php
// ✅ 正确：mock/real 切换 + 错误处理 + 日志
public function all(): array
{
    return config('integration.xxx_source', 'mock') === 'real'
        ? $this->allReal()
        : $this->allMock();
}

protected function allReal(): array
{
    $pdo = MySqlClient::pdo();
    if (!$pdo) {
        error_log("[XxxRepository] Database connection failed");
        return [];
    }
    try {
        $stmt = $pdo->query('SELECT ... FROM xxx ORDER BY id DESC');
        return $stmt->fetchAll(\PDO::FETCH_ASSOC) ?: [];
    } catch (\PDOException $e) {
        error_log("[XxxRepository] Query failed: " . $e->getMessage());
        return [];
    }
}
```

**必须遵守**：
1. 所有 PDO 操作包裹在 try-catch 中
2. 失败时记录 error_log，包含 Repository 名称
3. 支持 mock/real 切换

### 1.4 Middleware 规范

```php
<?php
namespace app\middleware;

use Webman\MiddlewareInterface;
use Webman\Http\Response;
use Webman\Http\Request;

class XxxMiddleware implements MiddlewareInterface
{
    // ✅ 必须实现 MiddlewareInterface
    // ✅ 参数类型必须是 Webman\Http\Request 和 Webman\Http\Response
    public function process(Request $request, callable $handler): Response
    {
        // 前置逻辑
        $response = $handler($request);
        // 后置逻辑
        return $response;
    }
}
```

### 1.5 命名规范

| 类别 | 规范 | 示例 |
|------|------|------|
| Controller | 大驼峰 + Controller 后缀 | `QuestionController` |
| Service | 大驼峰 + Service 后缀 | `QuestionAdminService` |
| Repository | 大驼峰 + Repository 后缀 | `QuestionRepository` |

| 配置键 | 蛇形 | `auth_rbac_source` |
| JSON 字段 | 蛇形 | `page_size`, `created_at` |

## 3. Git 规范

### 3.1 分支模型
```
main        ← 生产分支（只接受 merge）
  └── develop   ← 开发分支
       ├── feature/xxx   ← 功能分支
       ├── fix/xxx       ← 修复分支
       └── refactor/xxx  ← 重构分支
```

### 3.2 Commit 规范
```
<type>: <subject>

类型：
  feat:     新功能
  fix:      修复 bug
  refactor: 重构（不改功能不修 bug）
  docs:     文档
  style:    格式（不影响逻辑）
  test:     测试
  chore:    构建/工具/依赖

示例：
  feat: 添加搜题接口额度检查
  fix: 修复 DashboardController 未调用 Service 层
  refactor: 重构路由配置，使用路由分组
  docs: 重构项目文档体系
```

## 5. Mock → Real 切换路线

| 顺序 | 模块 | 配置键 | 涉及 Repository |
|------|------|--------|-----------------|
| 1 | 认证/RBAC | `auth_rbac_source` | User, Role, Permission, Menu |
| 2 | 题目/搜索 | `question_source` | Question(Mongo), QuestionIndex(ES) |
| 3 | 日志/额度 | `log_source` | SearchLog, Quota(Redis) |
| 4 | 用户中心 | `user_center_source` | Wallet, Subscription, ApiKey, Announcement |
| 5 | 文档 | `docs_source` | DocCategory, DocArticle, DocConfig |
| 6 | 采集 | `collect_source` | CollectTask, CollectAccount |
| 7 | 系统配置 | `config_source` | SystemConfig |
| 8 | API 源 | `api_source_source` | ApiSource |

**切换原则**：
1. 一次只切一个模块
2. 切换前确保数据库已建表、有 seed 数据
3. 切换后运行对应的冒烟测试
4. 验证通过后删除对应的 mock JSON 文件
