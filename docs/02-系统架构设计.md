## 4. 数据源切换机制

项目通过 `config/integration.php` 实现 mock 与 real 数据源的无缝切换。

```php
// config/integration.php
return [
    'auth_rbac_source'   => env('AUTH_RBAC_SOURCE', 'mock'),
    'question_source'    => env('QUESTION_SOURCE', 'mock'),
    'user_center_source' => env('USER_CENTER_SOURCE', 'mock'),
    'log_source'         => env('LOG_SOURCE', 'mock'),
    'docs_source'        => env('DOCS_SOURCE', 'mock'),
    'collect_source'     => env('COLLECT_SOURCE', 'mock'),
    'config_source'      => env('CONFIG_SOURCE', 'mock'),
    'api_source_source'  => env('API_SOURCE_SOURCE', 'mock'),
];
```

Repository 内部实现：
```php
class UserRepository
{
    public function findByUsername(string $username): array
    {
        return config('integration.auth_rbac_source') === 'real'
            ? $this->findByUsernameReal($username)    // MySQL 查询
            : $this->findByUsernameMock($username);   // JSON 文件
    }
}
```

**切换步骤**：
1. 确保对应数据库已建表、有数据
2. 修改 `.env` 中对应的 `*_SOURCE=real`
3. 重启 Webman

## 5. 路由与中间件设计

```php
// 公共路由（无需认证）
Route::get('/health', [HealthController::class, 'health']);

// 认证路由（无需认证）
Route::post('/api/v1/auth/login', [AuthController::class, 'login']);

// 用户端路由（需 JWT 认证）
Route::group('/api/v1/user', function () {
    Route::get('/dashboard/overview', [DashboardController::class, 'overview']);
    Route::post('/search/query', [SearchController::class, 'query']);
    // ...
})->middleware([UserAuthMiddleware::class]);

// 管理端路由（需 JWT + admin 角色）
Route::group('/api/v1/admin', function () {
    Route::get('/question/list', [QuestionController::class, 'index']);
    // ...
})->middleware([AdminAuthMiddleware::class]);

// 开放平台路由（需 API Key）
Route::post('/open/v1/search/query', [SearchController::class, 'query']);
```

## 6. 统一响应格式

所有 API 接口统一返回以下 JSON 格式：

```json
{
    "code": 1,
    "msg": "success",
    "data": {},
    "request_id": "req_xxxxxxxxxxxx"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 1=成功，40001=参数错误，40002=未登录，40003=无权限，50001=系统错误 |
| msg | string | 提示信息 |
| data | object/array | 业务数据 |
| request_id | string | 请求追踪 ID |

### 列表接口统一返回：
```json
{
    "code": 1,
    "msg": "success",
    "data": {
        "list": [],
        "total": 0,
        "page": 1,
        "page_size": 20
    },
    "request_id": ""
}
```

### 写操作统一返回：
```json
{
    "code": 1,
    "msg": "创建成功",
    "data": {
        "success": true,
        "action": "create",
        "id": 1,
        "data": {}
    },
    "request_id": ""
}
```

## 7. 异常处理

```
异常发生
    │
    ▼
support/exception/Handler（实现 ExceptionHandlerInterface）
    │
    ├─ report()：记录错误日志到 runtime/logs/
    │
    └─ render()：返回统一 JSON 响应
        ├─ app.debug=true  → 返回详细错误（exception, file, line, trace）
        └─ app.debug=false → 返回通用错误（"服务器内部错误"）
```

## 9. 性能架构

### 9.1 Webman 常驻内存

```
启动 → Workerman Master 进程
           │
           ├── Worker 1（HTTP Server）
           ├── Worker 2（HTTP Server）
           ├── Worker 3（HTTP Server）
           └── Worker N（HTTP Server）
                  │
                  │  每个 Worker 独立处理请求
                  │  无需每次请求重新加载框架
                  │  QPS 可达 10000+
```

- Worker 数量 = `cpu_count() * 4`
- 控制器不复用（`controller_reuse = false`），安全优先
- 每个 Worker 独立维护数据库连接

### 9.2 多级缓存

```
请求 → Redis L1 缓存（搜索结果、权限、额度）
  │       │
  │  命中 → 直接返回（<1ms）
  │       │
  │  未命中 → Elasticsearch L2（搜索索引）
  │              │
  │         命中 → 返回 + 写入 Redis
  │              │
  │         未命中 → MongoDB L3（原始数据）
  │                    │
  │               命中 → 返回 + 写入 ES + Redis
  │                    │
  │               未命中 → 404
```

### 9.3 数据库职责分离

| 存储 | 用途 | 优势 |
|------|------|------|
| MySQL | 结构化业务数据（用户、订单、配置） | 事务支持、关系查询 |
| MongoDB | 文档型数据（题目、日志详情） | 灵活 Schema、大字段存储 |
| Elasticsearch | 全文搜索索引 | 毫秒级全文检索 |
| Redis | 缓存、限流、额度、Token | 内存操作、原子性 |
