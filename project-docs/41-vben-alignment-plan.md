# Vben Admin 对齐与前端收口计划

## 一、当前项目与 Vben 官方理念一致的点
- 一个前端项目承载用户端与管理端
- 一个统一登录页
- 统一身份体系
- 菜单、权限、角色前置
- 表格页 / 卡片页 / 管理页壳子已经成型

## 二、当前与官方 Vben 5 仍有差距的点
- 当前不是官方 monorepo 结构
- 当前尚未正式引入 Vben 的完整布局与生态封装
- 当前权限守卫与菜单驱动还较轻量
- 组件层仍偏“骨架页”而不是统一 UI 封装组件页

## 三、当前建议
### 不建议现在做的事
- 不建议为了贴官方结构而推倒当前前端骨架
- 不建议在未开始真实部署前强行迁移到完整 monorepo

### 建议继续做的事
- 继续坚持统一登录 + 动态菜单 + 权限驱动页面
- 继续把管理页和用户页做成统一后台风格
- 在后续真实接入阶段，再逐步把布局、菜单、表格组件等往更官方的 Vben 风格靠拢

## 四、后续收口方向
1. layout 统一
2. auth store 统一
3. menu rendering 统一
4. page form/table 统一封装
5. 再评估是否需要完整接入官方 Vben 5 结构
