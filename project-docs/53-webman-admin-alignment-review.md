# webman-admin 对齐审查

## 当前判断
后端现在应遵循：
1. **运行骨架按 Webman 官方**
2. **后台业务层按 webman-admin 官方思路**

## 当前项目已符合的部分
- 统一用户体系（users + roles + permissions + menus）
- 认证/权限/菜单已独立成主线
- 管理端已经形成：用户/角色/权限/菜单/公告/系统配置/题目等模块

## 当前还可以继续按 webman-admin 收口的部分
- 后台 CRUD 模块进一步标准化
- 管理端页面接口命名与控制器职责进一步统一
- MySQL 业务优先使用更稳定的后台组织方式，而不是继续发散自定义层次

## 当前不建议推倒的部分
- 搜题主线（Mongo/ES/Redis 多数据源）
- 采集外围与未来采集脚本链
- 开放平台查询接口

## 结论
对于当前项目：
- **后台管理类模块**继续向 webman-admin 官方思路收口
- **搜题/采集/开放平台**保留项目自己的扩展设计
